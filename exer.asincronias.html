Asincronias & Promesas

¿Qué es un archivo JSON?

JSON significa Notación de objetos de JavaScript y es un formato utilizado para estructurar y transferir datos en forma de texto de un sistema a otro, como cliente/servidor y aplicaciones móviles.

¿Qué es la asincronía en Javascript?

La asincronía en Javascript se refiere a procesos que no sabemos cuánto tiempo tardarán en completarse.

¿Qué son las Promesas?

Las Promesas son un mecanismo que JavaScript nos ofrece para gestionar procesos asíncronos. Pueden ocurrir tres cosas:

  • La promesa se cumple (promesa resuelta).
  • La promesa no se cumple (promesa rechazada).
  • La promesa queda en un estado incierto indefinidamente (promesa pendiente).

¿Qué es y cómo se usa 'Fetch' en JS?

'Fetch' es una función integrada en JavaScript que se utiliza para realizar solicitudes de red y recuperar recursos del servidor. Se usa normalmente para enviar solicitudes HTTP a las API y recibir los datos correspondientes. Básicamente, se trata de llamar a 'fetch' y pasarle por parámetro la URL de la petición a realizar:

const promise = fetch("/robots.txt");
promise.then(function(response) {
    // ...
});

¿Cómo se usa Async/Await?

Async/Await es una característica introducida en JavaScript que proporciona una forma más limpia y legible de trabajar con promesas. El código asíncrono se puede escribir de forma síncrona, lo que hace que el flujo de control sea más fácil de entender y trabajar con él.

Para usar async/await, primero debes definir una función con la palabra clave 'async'. Esta función se convierte automáticamente en una función asíncrona que devuelve una promesa. Si usas 'await', debes estar dentro de una función asíncrona.

Es importante tener en cuenta que async/await está disponible en navegadores modernos y versiones actualizadas de Node.js.

¿Cuándo deberíamos usar código asíncrono?

El código asíncrono se usa cuando necesitamos realizar una operación que lleva tiempo sin bloquear la ejecución del programa. Esto permite que el hilo principal de ejecución no se bloquee por tareas que pueden llevar mucho tiempo, lo que resulta en una aplicación más eficiente y receptiva.
